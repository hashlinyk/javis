# 超清畅联平台设计和一期计划

## 一、项目概述

### 产品定位

**超清畅联**是一个文化内容聚合平台，作为用户与大屏超清体验之间的连接纽带。当用户想要在电视大屏上欣赏博物馆、画作、音视频、使用AI等文化内容时，超清畅联是首选平台。

- **内容聚合**：公网内容 + 专网内容
- **核心价值**：将家庭电视大屏转变为家庭美术馆、数字文博馆、文化消费中心
- **战略意义**：跳出传统广电思维，重塑大屏主流地位

### 终端交付物

- **手机端**：微信小程序（中广数科已有小程序，已申请工具类别，到时候可直接替换）
- **大屏端**：Android APK

## 二、核心交互逻辑

### 连接与首页机制

**进入小程序时的连接逻辑**：

```
用户打开小程序
    ↓
检测是否可连接大屏
    ↓
    ├─ 可连接 → 自动连接大屏
    │            ├─ 大屏显示首页（板块列表）
    │            └─ 小程序显示首页（板块列表）
    │
    └─ 不可连接 → 小程序正常显示首页（板块列表）
```

**核心原则**：小程序显示首页板块列表，大屏也显示首页板块列表，保持同步。

### 大小屏协同机制

| 状态              | 小程序页面       | 大屏页面                  | 用户行为                                            |
| ----------------- | ---------------- | ------------------------- | --------------------------------------------------- |
| **未连接**        | 首页（板块列表） | 无                        | 打开板块 → 在小程序内浏览                           |
| **已连接-首页**   | 首页（板块列表） | 首页（板块列表）          | 点击小程序板块 → 大屏进入资源列表，小程序跳转触摸板 |
| **已连接-板块内** | 触摸板页面       | 板块资源列表页/资源详情页 | 触摸板操作控制大屏                                  |

### 输入方式支持

1. **手机触摸板**（主要交互方式）
   - 强交互、个性化操作的核心输入设备
   - 支持触控映射：单指滑动、双指捏合/张开、双指拖拽等手势

- 支持光标模式

2. **传统遥控器**
   - 支持传统用户需求，不用手机也能操作
   - 例如千亿像素板块支持：+/-键缩放、方向键移动视角

### 页面状态流转

**已连接状态下的完整流程**：

```
小程序首页 + 大屏首页
    ↓
用户点击小程序首页的某个板块（如"文物日历"）
    ↓
大屏进入该板块的资源列表页
小程序跳转到触摸板页面
    ↓
用户在触摸板操作
    ├─ 单指滑动/点击 → 切换选中资源 / 进入资源详情页
    ├─ 点击"返回"按钮 → 大屏返回上一页
    └─ 点击"返回首页"按钮 → 大屏直接返回首页
    ↓
当大屏返回到首页时，自动通知小程序返回首页
    ↓
回到小程序首页 + 大屏首页
```

**触摸板按钮说明**：

**设计原则**：

- 触摸板保持精简，只保留必要的导航按钮
- 提供逐级返回和快捷返回两种方式，平衡操作效率和闭环完整性

**返回按钮**：

- **视觉样式**：贴着屏幕右侧的半椭圆形按钮
- **设计原因**：
  - 位置符合用户单手操作习惯，便于右手拇指点击
  - 半椭圆形设计增大点击区域，提升操作准确性
  - 与系统返回键位置接近，降低学习成本
- **触发方式**：
  - 点击触摸板上的返回按钮
  - 按手机系统物理返回键（等同于点击返回按钮）
- **功能逻辑**：
  - 通知大屏返回上一页
  - 当大屏从资源列表页返回到首页时，触发大屏首页回传机制

**返回首页按钮**：

- **视觉样式**：触摸板页面左上角的房子图标
- **设计原因**：
  - 房子图标是通用的"首页"符号，用户易于理解
  - 位于左上角，与返回按钮形成对角布局，避免误触
  - 提供快捷退出通道，无需逐级返回，提升操作效率
- **触发方式**：点击触摸板上的房子图标
- **功能逻辑**：
  - 点击后大屏直接返回首页（板块列表）
  - 大屏返回首页后，自动触发大屏首页回传机制
  - 小程序收到通知后，自动返回小程序首页
  - 无论大屏处于资源列表页还是详情页，均可响应

**大屏首页回传机制**：

- 当大屏返回到首页时，主动回传数据通知小程序
- 小程序收到通知后，自动返回小程序首页

**页面流转示例**：

```
小程序首页 + 大屏首页（板块列表）
    ↓ [用户点击板块]
小程序触摸板 + 大屏资源列表页
    ↓ [用户点击选中资源]
小程序触摸板 + 大屏资源详情页
    ↓ [点击"返回"]
小程序触摸板 + 大屏资源列表页
    ↓ [点击"返回"]
小程序首页 + 大屏首页（板块列表）[大屏自动通知小程序]

【快捷方式】
小程序触摸板 + 大屏资源详情页
    ↓ [点击"返回首页"]
小程序首页 + 大屏首页（板块列表）[大屏自动通知小程序]
```

### 状态同步机制

- **大屏 → 手机**：大屏数据回传，触发手机端执行相应的逻辑，例如：
  - 大屏返回首页时，数据回传，通知小程序返回首页（使小程序自动从触摸板回到首页），保持首页同步
  - 小程序触摸板动态显示当前大屏场景对应快捷操作按钮（如大屏有“数字藏品”标记时，手机触摸板显示交易按钮）

### 大屏遥控器支持

**支持范围**：大屏首页支持遥控器操作

**操作方式**：

- 使用遥控器方向键选择板块
- 按确认键进入板块
- 大屏进入板块资源列表页后，小程序自动跳转到触摸板页面

**使用场景**：

- 用户习惯使用遥控器操作大屏
- 手机不在身边或不需要使用手机触摸板
- 提升大屏端的主动性和独立性

### 多用户操作

- 每个用户的操作状态需要独立记录，典型场景：不同用户在同一个板块上次浏览的内容不同，下次进入板块时希望焦点默认位于不同的资源上
- 支持多个用户（多手机）同时操作同一个大屏，典型场景：多人游戏（如棋牌类游戏，两个人各自用手机操作，在大屏对弈）

#### 3.1 完整的大小屏交互能力

实现完整的触摸板交互功能，支持触控映射和光标两种模式，提供返回和返回首页按钮，实现大屏与手机的状态同步。

#### 3.2 文物日历

**未连接大屏时**：在小程序内浏览文物列表和详情页。

**已连接大屏时**：用户点击小程序首页的"文物日历"板块，大屏进入资源列表页，小程序跳转到触摸板页面。

**触摸板操作**：

- 资源列表页：单指上下滑动切换选中文物，单指点击进入文物详情页
- 文物详情页（3D模型）：单指滑动旋转视角，双指捏合/张开缩放模型

#### 3.3 中华珍宝馆

**未连接大屏时**：在小程序内浏览艺术藏品墙和画作详情页。

**已连接大屏时**：用户点击小程序首页的"中华珍宝馆"板块，大屏进入艺术藏品墙（资源列表页），小程序跳转到触摸板页面。

**触摸板操作**：

- 资源列表页（艺术藏品墙）：单指左右滑动切换选中画作，单指点击进入画作详情页
- 画作详情页（超高清图像）：单指滑动平移画面，双指捏合/张开无级缩放（瓦片加载），双指拖拽检查细节

#### 3.4 千亿像素看中国

**未连接大屏时**：在小程序内浏览地标列表和详情页。

**已连接大屏时**：用户点击小程序首页的"千亿像素看中国"板块，大屏进入地标列表页，小程序跳转到触摸板页面。

**触摸板操作**：

- 资源列表页：单指上下滑动切换选中地标，单指点击进入地标详情页
- 地标详情页（全景漫游）：
  - 带热点的全景：带有热点的全景会默认显示光标。单指滑动移动大屏光标，光标热点点击触发场景穿越或介绍，双指同向滑动拖拽视角，双指捏合/张开深度缩放（瓦片加载）
  - 不带热点的全景：单指滑动/双指同向滑动拖拽视角，双指捏合/张开深度缩放（瓦片加载）

### 第二阶段：4月15日 融合AI语音助手基础功能

在上一个版本基础上，三大板块均融合**AI语音助手**，实现导览介绍、解答用户疑问的交互体验。

> 注：触摸板底部显示语音按钮，长按触发AI语音助手

为每个文物、珍宝画作、地标设定一些物体介绍、背景故事、导览文案、热点标注等资料，当用户在微信小程序上用语音提出问题时，AI语音助手可以在大屏端回应，实现导览介绍、解答用户疑问

### 4.4 AI 识图（核心）

在用户查看某个文物或者画作、景点等3D素材的局部图像细节时，如果用户向AI助手发起询问，AI助手可以针对当前屏幕显示的内容进行针对性的答疑介绍等。

**方案一：训练专有模型**

- 训练专有模型，用于识别各种文旅艺术类的图像资源
- 优势：识别准确度高，适用范围广
- 劣势：需要大量素材标注数据，时间长、成本高、投入大

**方案二：隐形热点 + AI**

- 使用隐形热点 + 标注文字资料 + AI
- 触发时统计当前屏幕所有热点（含隐形热点），再交由AI分析
- 优势：可变相实现"识图"的效果，成本低、实现快
- 劣势：需要提前标注热点，适用范围受限

### 4.5 用户体验优化需求

**交互优化**：

- 物体热点弹出介绍后，点击左右箭头操作路径长 → 可实现触摸板单指滑动切换效果
- 缩放位置偏差大 → 优化触控映射精度

**状态记忆与推荐**：

- 进入列表页或者详情页时，回到上次资源访问时的位置或缩放位置
- 每个用户的操作状态需要独立记录
- 在长列表中滑动切换不同项目时，显示进度（如 99/1000）
- 优先推荐用户感兴趣的资源

### 4.6 多用户操作同一个大屏

**客户现有实现方案**：

- **连接方式**：大小屏连接同一局域网WiFi
- **配对流程**：
  1. 大屏首页右下角显示二维码
  2. 用户用手机扫描大屏二维码
  3. 扫码后完成连接，手机可操作大屏
- **多手机支持**：已实现多用户（多手机）同时操作同一个大屏
